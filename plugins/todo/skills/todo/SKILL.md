# Todo管理スキル

Todoファイル: `~/.todo/todos.md`

まず必ずこのファイルを読んでから処理する。

## 番号体系

表示時に動的採番する（ファイルには番号を保存しない）。

- トップレベルタスク: `1`, `2`, `3` ...
- 子タスク: `1.1`, `1.2`, `2.1` ...
- メモ: `1-1`, `1-2`（タスク番号-メモ番号）
- 子タスクのメモ: `1.1-1`, `1.1-2`

## 呼び出しパターン

### `help`
以下をそのまま表示する:

```
## Todo コマンド一覧

### タスク
/todo                          # 一覧表示・提案
/todo add 買い物               # タスク追加
/todo add 1 > 牛乳を買う       # 子タスク追加（番号指定）
/todo done: 1                  # タスク完了
/todo done: 1.2                # 子タスク完了
/todo rm: 1                    # タスクをゴミ箱へ
/todo rm: 1.2                  # 子タスクをゴミ箱へ

### メモ
/todo memo add: 1 内容         # タスク1にメモ追加
/todo memo add: 1.2 内容       # 子タスク1.2にメモ追加
/todo memo rm: 1-1             # タスク1のメモ1を削除
/todo memo rm: 1.2-1           # 子タスク1.2のメモ1を削除
```

### 引数なし（`/todo`）
1. 未完了タスクの一覧を番号付きで表示（メモ・子タスクもインデントして表示）
2. 完了タスク数を簡潔に表示
3. 今やるべきタスクを提案（登録順を基本に）

表示フォーマット:
```
1. [ ] ECサイト開発 (登録: 2026-05-21 10:00)
       1-1: APIキーはenvに入れること
       1-2: デプロイはstaging→prod順で
   1.1. [ ] ログイン機能実装
          1.1-1: OAuth対応が必要
   1.2. [ ] 商品一覧ページ
2. [ ] ブログ執筆 (登録: 2026-05-21 11:00)
```

### `add <タスク名>`
1. ファイルを読んで既存の未完了タスク一覧を把握する
2. 子タスク判定を行う（下記「子タスク判定ロジック」を参照）
3. 子タスクにすべきと判断した場合はユーザーに確認してから登録
4. 子タスクにしない場合はトップレベルタスクとして登録

### `add <親番号> > <タスク名>`
- 指定した番号の親タスクの子として登録する
- 親タスクが存在しない場合はエラーを伝える
- 確認は不要、そのまま登録する

### `done: <番号>`
- 番号でタスクを特定して完了セクションに移動
- トップレベル（`1`）も子タスク（`1.1`）も対応
- フォーマット: `- [x] タスク名 (登録: YYYY-MM-DD HH:MM, 完了: YYYY-MM-DD HH:MM)`
- 登録時刻はそのまま保持し、完了時刻を追記
- メモも一緒に移動する
- 子タスクを完了させた場合、未完了の兄弟タスクがあれば次の子タスクを提案する
- 親タスクの完了は手動（自動完了しない）

### `rm: <番号>`
- 番号でタスクを特定してゴミ箱セクションに移動
- トップレベル（`1`）の場合は子タスク・メモも含めて移動
- 子タスク（`1.1`）の場合はその子タスクとそのメモのみ移動
- フォーマットはそのまま保持（`[ ]` のまま移動）

### `memo add: <番号> <内容>`
- 指定した番号のタスクにメモを追加（トップレベル・子タスク両対応）
- 既存メモがあっても追記（上書きではない）

### `memo rm: <メモ番号>`
- メモ番号形式: `1-1`（タスク1のメモ1）、`1.1-2`（タスク1.1のメモ2）
- 該当メモ行をファイルから削除する

## 子タスク判定ロジック

`add <タスク名>` で親指定がない場合、以下の基準で判定する。

**子タスクと判断するケース：**
- 新タスクが既存タスクの具体的な実装ステップ・サブ作業に見える
- 新タスクの内容が既存タスクのスコープに明確に含まれる
- 例：「ECサイト開発」があるときに「ログイン機能実装」を登録 → 子タスク候補

**トップレベルと判断するケース：**
- 既存タスクと独立した別の目的・テーマを持つ
- 関連はあっても並列関係（同レベルの作業）
- 既存タスクが1件以下しかない

**判定後の動作：**
- 子タスク候補が1件見つかった場合：「`<親タスク名>` の子タスクとして登録しますか？」と確認
- 候補が複数ある場合：候補をリストアップして選ばせる（「どの親タスクの子にしますか？それともトップレベル？」）
- 子タスクにしない場合：トップレベルに登録

## ファイルフォーマット

```markdown
# Todo

## 未完了
- [ ] タスク名 (登録: 2026-05-21 10:00)
  > メモ1
  > メモ2
  - [ ] 子タスク名 (登録: 2026-05-21 11:00)
    > 子タスクのメモ
  - [ ] 子タスク名2 (登録: 2026-05-21 11:30)
- [ ] 別タスク (登録: 2026-05-21 12:00)

## 完了
- [x] タスク名 (登録: 2026-05-21 09:00, 完了: 2026-05-21 12:00)
  > メモ（完了後も保持）
  - [x] 子タスク名 (登録: 2026-05-21 09:30, 完了: 2026-05-21 11:00)

## ゴミ箱
- [ ] 削除したタスク (登録: 2026-05-21 08:00)
  > メモ（ゴミ箱移動後も保持）
  - [ ] 削除した子タスク (登録: 2026-05-21 08:30)
```

- メモ行はタスク行の直後に `  > ` プレフィックス（2スペース + `> `）で記述
- 子タスクのメモは子タスク行の直後に `    > ` プレフィックス（4スペース + `> `）で記述
- 子タスクは親タスクの直下に2スペースインデントで記述
- ファイルが存在しない場合は上記フォーマットで新規作成する
