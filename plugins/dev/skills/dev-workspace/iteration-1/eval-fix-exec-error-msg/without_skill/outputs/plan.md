# Plan - fix-exec-error-msg

## 概要

`/dev exec` 実行時に `tasks.md` が存在しない場合、SKILL.md にエラーハンドリングの定義がないため LLM が不明瞭な挙動をする。

SKILL.md の `/dev exec` ステップ1に tasks.md 不在チェックと明確なエラーメッセージテンプレートを追記し、ユーザーが原因と次の行動を即座に理解できるようにする。

## 実装方針

SKILL.md はコードではなくLLMへの指示書（Markdown）のため、修正もMarkdown追記のみ。

`/dev exec` ステップ1に以下を追加する:

1. **不在チェックの指示**: `tasks.md` が存在しない場合は処理を中断してエラーメッセージを表示するよう記述
2. **エラーメッセージテンプレート**: 表示する内容を明示（原因・対処法を含む）

表示するエラーメッセージのイメージ:

```
エラー: tasks.md が見つかりません（~/.dev/<project>/<feature>/tasks.md）

考えられる原因:
  - /dev plan がまだ実行されていない
  - /dev plan の途中で中断した（plan.md のみ存在する状態）

対処法:
  1. /dev plan を実行して計画ファイルを生成してください
  2. plan.md のみ存在する場合は /dev plan のステップ6（アウトプット生成）から再開してください
```

## 影響範囲

| ファイル | 変更種別 | 内容 |
|---------|---------|------|
| `plugins/dev/skills/dev/SKILL.md` | 追記 | `/dev exec` ステップ1に tasks.md 不在チェックとエラーメッセージテンプレートを追加 |

## 考慮事項

- **後方互換性**: 追記のみのため既存の正常系フローに影響なし
- **スコープ限定**: plan.md・investigation/ の不在ケースは今回対象外
- **LLM指示書としての記述**: コードではないため「〜の場合は〜を表示して中断する」という自然言語の指示で実現する
