# Plan - dev-review-cmd

## 概要

devスキルに `/dev review` コマンドを追加する。
`/dev exec` で実装したコードをAIがレビューし、問題点・改善提案を構造化して出力するコマンド。
開発者がセルフレビューの代わりにAIへ投げることで、コードレビューの品質担保と手戻りを削減する。

## 実装方針

SKILL.md に `review` セクションを追記する。既存の `plan` / `exec` セクションと同構造（ステップ番号付き）で記述。

### reviewコマンドのフロー

1. **対象の特定**: git diffでレビュー対象を取得。ブランチ・コミット範囲はユーザーが任意指定。
2. **コンテキスト収集**: フィーチャー名を確認し、`plan.md` / `tasks.md` があれば読み込んで文脈に活用。
3. **レビュー実行**: 取得コードを以下の観点でレビュー:
   - 正確性（バグ・ロジックエラー）
   - セキュリティ（インジェクション・認証漏れ等）
   - パフォーマンス（N+1・不要な処理等）
   - 可読性・保守性（命名・複雑度等）
   - テスト漏れ
4. **結果出力**: 重大度別（Critical / Warning / Suggestion）に整理してユーザーに提示し、`review.md` に保存。

## 影響範囲

| ファイル | 変更内容 |
|---------|---------|
| `plugins/dev/skills/dev/SKILL.md` | `review` コマンドセクション追記（description更新も含む） |
| `plugins/dev/.claude-plugin/plugin.json` | description に `review` コマンドを追記 |

## 考慮事項

- **後方互換性**: 既存の `plan` / `exec` セクションは変更しない
- **git diff 依存**: git管理外のプロジェクトでは動作しない（エラーメッセージで案内）
- **大きな差分**: 差分が大きい場合はファイル単位で分割レビューを推奨する旨を明示
- **フィーチャー未紐付けの場合**: `plan.md` がなくても単体で動作可能にする

## トレーサビリティ

| 要件 | コンポーネント | API | データ |
|------|--------------|-----|--------|
| コードレビュー実行 | SKILL.md `review` セクション | - | git diff |
| 問題点の構造化出力 | reviewコマンド ステップ3-4 | - | review.md |
| 計画との整合性確認 | reviewコマンド ステップ2 | - | plan.md / tasks.md |
